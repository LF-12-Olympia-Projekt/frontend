# Contributing

## Branch-Naming

Verwende das folgende Branch-Schema:

- `feature/<ISSUE-ID>-<kurzbeschreibung>`

## Commit-Konvention

Commits folgen dem Schema:

- `feat(scope): message`
- `fix(scope): message`
- `chore(scope): message`

## Pull Requests

- Jeder Pull Request benötigt vor dem Merge ein Review von mindestens einer weiteren Person (Vier-Augen-Prinzip).
- Die CI muss vor dem Merge vollständig grün sein.
